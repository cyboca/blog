Annotations Nginx Ingress

 发表于 November 17, 2019  |  1 分钟  |  119 个字  |  emiya

nginx ingress功能测试说明

[TOC]

1、ingress.class

  1. 格式：kubernetes.io/ingress.class: “nginx”
  2. 说明：设置ingress 类型

2、use-regex

  1. 格式：nginx.ingress.kubernetes.io/use-regex: “true”

  2. 说明：开启use-regex，启用path的正则匹配

3、app-root

  1. 格式：nginx.ingress.kubernetes.io/app-root string

  2. 功能：重定向到指定目录

4、rewrite-target

  1. 格式：nginx.ingress.kubernetes.io/rewrite-target: path/url

  2. 说明:重写规则、支持正则表达式

  3. 示例：

nginx.ingress.kubernetes.io/rewrite-target: https://www.baidu.com

nginx.ingress.kubernetes.io/rewrite-target: /

5、configuration-snippet

  1. 格式：nginx.ingress.kubernetes.io/configuration-snippet: |

  2. 说明：支持nginx规则

  3. 示例：

nginx.ingress.kubernetes.io/configuration-snippet: | rewrite ^/(.) http://www.baidu.com/ nginx.ingress.kubernetes.io/configuration-snippet: | rewrite /ana /animal/; nginx.ingress.kubernetes.io/configuration-snippet: | rewrite ^/animal(/|$)?(.) /$2/ break;

6、server-alias

  1. 格式：nginx.ingress.kubernetes.io/server-alias: “,“

  2. 说明：服务器别名

  3. 示例：

nginx.ingress.kubernetes.io/server-alias: “nginx.aa.com”

7、limit-connections

  1. 格式：nginx.ingress.kubernetes.io/limit-connections: number

  2. 说明：单个IP地址允许的并发连接数。超出此限制时，将返回503错误

  3. 示例：

nginx.ingress.kubernetes.io/limit-connections: “11”

8、limit-rps、limit-rpm

  1. 格式:nginx.ingress.kubernetes.io/limit-rps: number

  2. 说明：用来限制单位时间内的请求数，即速率限制,采用的漏桶算法 “leaky bucket”；每秒从给定IP接受的请求数。突发限制设置为限制的5倍。当客户端超过此限制时，将返回503

  3. 示例：

nginx.ingress.kubernetes.io/limit-rps: “2”

  1. 参数说明：

limit_req_zone $binary_remote_addr zone=one:10m rate=1r/s;

  • $binary_remote_addr 表示通过remote_addr这个标识来做限制，“binary_”的目的是缩写内存占用量，是限制同一客户端ip地址。

  • zone=one:10m表示生成一个大小为10M，名字为one的内存区域，用来存储访问的频次信息。

  • rate=1r/s表示允许相同标识的客户端的访问频次，这里限制的是每秒1次，还可以有比如30r/m的

limit_req zone=one burst=5 nodelay;

  • zone=one 设置使用哪个配置区域来做限制，与上面limit_req_zone 里的name对应。

  • burst=5，重点说明一下这个配置，burst爆发的意思，这个配置的意思是设置一个大小为5的缓冲区当有大量请求（爆发）过来时，超过了访问频次限制的请求可以先放到这个缓冲区内。

  • nodelay，如果设置，超过访问频次而且缓冲区也满了的时候就会直接返回503，如果没有设置，则所有请求会等待排队

9、limit-rate

  1. 格式：nginx.ingress.kubernetes.io/limit-rate-after
  2. 说明：初始千字节数，在此之后对给定连接的响应的进一步传输将受到速率限制。
  3. 格式：nginx.ingress.kubernetes.io/limit-rate
  4. 说明：每秒发送到给定连接的千字节数。零值禁用速率限制。
  5. 格式：nginx.ingress.kubernetes.io/limit-whitelist
  6. 说明：客户端IP源范围要从速率限制中排除。该值是逗号分隔的CIDR列表。

10、temporal-redirect

  1. 格式：nginx.ingress.kubernetes.io/temporal-redirect：

  2. 说明：此注释使您可以返回重定向（返回代码302），而不是将数据发送到上游

  3. 示例：

nginx.ingress.kubernetes.io/temporal-redirect：http://www.baidu.com

11、proxy-buffering

  1. 格式：nginx.ingress.kubernetes.io/proxy-buffering: string

  2. 说明：开启代理缓冲（默认关闭）

  3. 示例：

nginx.ingress.kubernetes.io/proxy-buffering: “on”

12、proxy-buffers-number

  1. 格式：nginx.ingress.kubernetes.io/proxy-buffers-number

  2. 说明：设置缓冲区数

  3. 示例：

nginx.ingress.kubernetes.io/proxy-buffers-number: “4”

13、proxy-buffer-size

  1. 格式：nginx.ingress.kubernetes.io/proxy-buffer-size: string

  2. 说明：设置缓冲区大小

  3. 示例：

nginx.ingress.kubernetes.io/proxy-buffer-size: “8k”

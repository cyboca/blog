NetWorkPolicy

NetworkPloicy即网络策略（访问控制）；定义容器组(pod)与其他容器组以及endpoint进行通信；networkPolisy资源使用pod标签来选择pod，并定义规则

networkpolicy的实现需要特定的网络组件的支持;

1、实验环境准备

如上图所示：新建两个命令空间ns1和ns2并且为两个命名空间分别添加两个标签；创建上图中的三和deployment（过程略）用于测试网络连通性

2、NetworkPolicy配置以及测试

如上图所示默认情况下相同命令空间下的pod和不同命名空间下的pod是可以相互访问的，如果需要做特定的访问控制需要用到networkpolicy这一资源

2.1 deny all

[root@node01 networkpolicy]# cat default-netpol.yaml 
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default
  namespace: ns1
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  ingress:
[root@node01 networkpolicy]# kubectl apply -f default-netpol.yaml 
networkpolicy.networking.k8s.io "default" created
[root@node01 networkpolicy]#

如上图所示ns1命令空间下的pod网络被隔离了

2.2 allow all(默认)

[root@node01 networkpolicy]# cat default-netpol.yaml 
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default
  namespace: ns1
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  ingress:
  - {}
[root@node01 networkpolicy]# kubectl apply -f default-netpol.yaml 
networkpolicy.networking.k8s.io "default" created
[root@node01 networkpolicy]#

如上图所示，ns1命名空间下的pod的网络没有被限制，

2.3针对namespace的访问控制

[root@node01 networkpolicy]# cat default-netpol.yaml 
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default
  namespace: ns1
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          name: ns1
[root@node01 networkpolicy]# kubectl apply -f default-netpol.yaml 
networkpolicy.networking.k8s.io "default" configured
[root@node01 networkpolicy]#

如上图所示处于同一命名空间下的pod能够访问，不通命令空间下的pod决绝访问，并且主机端也无法访问该命令空间下的pod，针对主机对pod的访问控制见下一小结

2.4针对ip的访问控制

[root@node01 networkpolicy]# cat default-netpol.yaml 
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default
  namespace: ns1
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          name: ns1
    - ipBlock:
        cidr: 192.168.10.0/24
[root@node01 networkpolicy]# kubectl apply -f default-netpol.yaml 
networkpolicy.networking.k8s.io "default" configured
[root@node01 networkpolicy]#

如果希望某一ip或ip段禁止访问可以使用except参数

[root@node01 networkpolicy]# cat default-netpol.yaml 
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default
  namespace: ns1
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          name: ns1
    - ipBlock:
        cidr: 192.168.10.0/24
        except:
        - 192.168.10.21/32
[root@node01 networkpolicy]# kubectl apply -f default-netpol.yaml 
networkpolicy.networking.k8s.io "default" configured
[root@node01 networkpolicy]#

如上图所示ip为192.168.10.21的node02节点访问pod失败；如果只希望pod的某一端口能被访问，可以使用对特定端口的访问控制，具体配置见下一小结

2.5针对pod特定端口的访问控制

[root@node01 networkpolicy]# cat default-netpol.yaml 
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default
  namespace: ns1
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          name: ns1
    - ipBlock:
        cidr: 192.168.10.0/24
        except:
        - 192.168.10.21/32
    ports:
    - protocol: TCP
      port: 80
[root@node01 networkpolicy]# kubectl apply -f default-netpol.yaml 
networkpolicy.networking.k8s.io "default" configured
[root@node01 networkpolicy]#

如上图所示ping命令执行失败，但容器的80端口是能够正常访问的

2.6备注

以上配置都是针对命令空间下的所有容器的ingress(入口)访问控制；如果希望针对容器出口流量的访问控制需要修改ingress为egress，修改from为to，当然两者也可以同时使用
